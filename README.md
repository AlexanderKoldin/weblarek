# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Vite

## Структура проекта:

- `src/` — исходные файлы проекта
- `src/components/` — папка с компонентами логики и отображения
- `src/components/base/` — папка с базовым кодом (API, брокер событий, базовый компонент)
- `src/components/Models/` — папка с моделями данных

## Важные файлы:

- `index.html` — HTML-файл главной страницы
- `src/types/index.ts` — файл с типами данных
- `src/main.ts` — точка входа приложения (Presenter)
- `src/scss/styles.scss` — корневой файл стилей
- `src/utils/constants.ts` — файл с константами (API_URL, CDN_URL)
- `src/utils/utils.ts` — вспомогательные функции для работы с DOM

## Интернет-магазин «Web-Larёk»

**Web-Larёk** — это интернет-магазин с товарами для веб-разработчиков.  
Пользователи могут:

- просматривать каталог
- добавлять товары в корзину
- оформлять заказы с выбором способа оплаты

## Архитектура приложения

Приложение реализовано в рамках паттерна **MVP (Model-View-Presenter)**:

- **Model** — отвечает за хранение и обработку данных  
  (`CatalogModel`, `BasketModel`, `OrderModel`)

- **View** — отвечает за визуальное отображение  
  (базовый класс `Component`)

- **Presenter** — связующее звено, реализованное в `src/main.ts`  
  Инициализирует модели, настраивает слушателей событий в `EventEmitter`  
  и координирует работу API и интерфейса. В слое **Presenter** также реализована логика «живой» фильтрации ввода для поля телефона: любые символы, кроме цифр и разрешенных знаков (`+`, `()`, `-`), удаляются автоматически на лету. Это гарантирует чистоту данных в модели и улучшает пользовательский опыт.

Взаимодействие слоев организовано через событийно-ориентированный подход  
с использованием брокера событий **EventEmitter**.

## Базовый код

### Класс Component<T>

Абстрактный базовый класс для всех компонентов интерфейса. Предназначен для управления DOM-элементами и отображения данных.

**Конструктор:** `constructor(container: HTMLElement)` — принимает корневой DOM-элемент.

**Методы:**

- `render(data?: Partial<T>): HTMLElement` — обновляет свойства компонента через сеттеры и возвращает его элемент.
- `setImage(element: HTMLImageElement, src: string, alt?: string): void` — устанавливает изображение и альтернативный текст.
- `setDisabled(element: HTMLElement, state: boolean): void` — блокирует/разблокирует элемент.
- `setText(element: HTMLElement, value: unknown): void` — устанавливает текстовое содержимое.
- `toggleClass(element: HTMLElement, className: string, force?: boolean): void` — переключает CSS-класс.

### Класс Api

Содержит базовую логику отправки запросов на сервер.

**Конструктор:**

```ts
constructor(baseUrl: string, options: RequestInit = {})
```

**Методы:**

- `get<T>(uri: string): Promise<T>` — GET запрос
- `post<T>(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<T>` — POST запрос
- `handleResponse<T>(response: Response): Promise<T>` — обработка ответа сервера

### Класс EventEmitter

Брокер событий (паттерн **«Наблюдатель»**).  
Обеспечивает связь между Моделями и Презентером.

**Методы:**

- `on` — подписка
- `emit` — уведомление
- `trigger` — создание триггера
- `off` — отписка
- `onAll` — слушать всё
- `offAll` — сброс

## Слой данных (Model)

### Типы данных (Interfaces)

**IProduct** — модель отдельного товара в магазине:

- `id: string` — уникальный идентификатор товара в системе (UUID);
- `description: string` — полное текстовое описание товара, отображаемое в модальном окне;
- `image: string` — URL-адрес изображения товара (относительный путь);
- `title: string` — краткое название товара;
- `category: string` — категория товара (например, «софт-скил», «хард-скил»);
- `price: number | null` — стоимость товара в «синапсах». Может быть `null`, если товар нельзя купить.

**IBuyer** — данные покупателя:

- `payment: PaymentMethod` — выбранный способ оплаты (`card` или `cash`);
- `address: string` — адрес доставки;
- `email: string` — контактная электронная почта;
- `phone: string` — контактный номер телефона.

**IOrder** — полный объект заказа для отправки на сервер:

- Все поля из `IBuyer`;
- `total: number` — итоговая сумма заказа;
- `items: string[]` — массив ID выбранных товаров.

**FormErrors** — объект с текстами ошибок валидации:

- `Partial<Record<keyof IBuyer, string>>` — ключами являются поля формы, значениями — текст ошибки.

## Модели данных

### Класс CatalogModel

Хранит каталог товаров и управляет их состоянием.

**Поля:**

- `_items: IProduct[]` — массив товаров.

**Методы и геттеры:**

- `setItems(items: IProduct[]): void` — записывает товары в модель и генерирует событие `items:changed`;
- `get items(): IProduct[]` — геттер для получения списка всех товаров.

### Класс BasketModel

Управляет состоянием корзины (добавление, удаление, расчет стоимости).

**Поля:**

- `_items: IProduct[]` — массив товаров, добавленных в корзину.

**Методы:**

- `add(item: IProduct): void` — добавляет товар и уведомляет систему через событие `basket:changed`;
- `remove(id: string): void` — удаляет товар по ID;
- `clear(): void` — полная очистка корзины;
- `getItems(): IProduct[]` — возвращает текущие товары в корзине;
- `getTotal(): number` — возвращает общую сумму всех товаров;
- `getCount(): number` — возвращает количество товаров в корзине;
- `hasItem(id: string): boolean` — проверяет, есть ли уже такой товар в корзине по его ID.

### Класс OrderModel

Отвечает за хранение данных покупателя, их проверку и хранение состояния ошибок.

**Поля:**

- `order: IBuyer` — объект, содержащий данные текущего заказа (способ оплаты, адрес, телефон, почта);
- `formErrors: FormErrors` — объект, хранящий текущие ошибки валидации форм.

**Методы:**

- `setField(field: keyof IBuyer, value: string): void` — устанавливает значение в поле заказа и запускает автоматическую валидацию;
- `validateOrder(): boolean` — проверяет данные покупателя (включая формат Email и телефона через регулярные выражения). Записывает ошибки в `formErrors`, уведомляет систему через событие `formErrors:change` и возвращает `true`, если данные корректны;
- `getOrderData(): IBuyer` — возвращает текущий объект данных заказа;
- `clearOrder(): void` — сбрасывает данные заказа и очищает объект ошибок.

## Слой представления (View)

Все компоненты представления наследуются от базового класса `Component<T>`. Они отвечают за отображение интерфейса, прием действий пользователя и генерацию событий через `EventEmitter`.

### Базовые компоненты

#### Класс Modal

Универсальное модальное окно. Управляет отображением контента, перекрытием страницы и блокировкой скролла.

- `set content(value: HTMLElement)` — устанавливает содержимое окна;
- `open(): void` — открывает модалку;
- `close(): void` — закрывает модалку.

#### Класс Form<T>

Базовый класс для работы с формами. Управляет валидацией и состоянием кнопки отправки.

- `set valid(value: boolean)` — управляет доступностью кнопки сабмита;
- `set errors(value: string)` — устанавливает текст ошибки в специальное поле;
- `render(state: Partial<T> & FormState)` — обновляет состояние формы (ошибки и валидность).

### Компоненты страницы

#### Класс Page

Управляет основными элементами главной страницы.

- `set counter(value: number)` — обновляет счетчик товаров на иконке корзины;
- `set catalog(items: HTMLElement[])` — рендерит массив карточек в галерее;
- `set locked(value: boolean)` — блокирует прокрутку страницы (при открытии модалок).

#### Класс Card

Универсальный компонент карточки товара. Имеет три специализации (каталог, превью, корзина).

- `set title(value: string)` — заголовок;
- `set image(value: string)` — изображение;
- `set price(value: number | null)` — цена (при `null` блокирует кнопку покупки);
- `set category(value: string)` — категория товара;
- `set button(value: string)` — текст кнопки (Купить/Удалить);
- `set index(value: string)` — порядковый номер карточки в корзине.

### Компоненты корзины и заказа

#### Класс Basket

Компонент корзины, отображает список выбранных товаров.

- `set items(items: HTMLElement[])` — обновляет список карточек товаров в корзине. Если корзина пуста, кнопка оформления блокируется;
- `set total(value: number)` — устанавливает итоговую сумму заказа.

#### Класс Order (наследует Form)

Форма выбора способа оплаты и ввода адреса.

- `set address(value: string)` — устанавливает значение адреса;
- `togglePayment(target: HTMLElement)` — переключает активную кнопку способа оплаты.

#### Класс Contacts (наследует Form)

Форма ввода контактных данных (email и телефон).

- `set email(value: string)` — устанавливает почту;
- `set phone(value: string)` — устанавливает телефон.

#### Класс Success

Окно подтверждения успешной оплаты.

- `set total(value: number)` — отображает итоговую списанную сумму;
- `render(data: ISuccess): HTMLElement` — метод отрисовки окна успеха с переданными данными.

## Взаимодействие слоев (Events)

Связь между компонентами Model и View осуществляется через события, которые обрабатываются в Presenter (main.ts).
Список основных событий:

- `items:changed` — изменение списка товаров в каталоге;
- `card:select` — выбор карточки товара для отображения в модальном окне;
- `basket:open` — открытие модального окна корзины;
- `basket:changed` — изменение списка товаров в корзине (обновление счетчика и суммы);
- `order:open` — переход к первому шагу оформления заказа (оплата и адрес);
- `order:submit` — переход ко второму шагу заказа (контактные данные);
- `formErrors:change` — событие, сообщающее об изменении состояния валидации обеих форм (заказа и контактов). На основе этого события Презентер управляет доступностью кнопок «Далее» и «Оплатить», блокируя их при наличии ошибок в соответствующих полях.
- `contacts:submit` — отправка данных сформированного заказа на сервер.

## Слой коммуникации (API)

### Класс LarekApi

Предоставляет методы для взаимодействия с серверной частью магазина.

**Методы:**

- `getLotList(): Promise<IProduct[]>` — получает полный список товаров с сервера;
- `orderLots(order: IOrder): Promise<IOrderResult>` — отправляет сформированный заказ на сервер и возвращает результат (ID и сумму).
